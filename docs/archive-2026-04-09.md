# Q-Vox-Lab Development Archive — 2026-04-09

## Project Overview

**Q-Vox-Lab** is a Flutter-based AI audio middleware and dubbing workstation.
- Platform: Windows / Android (responsive layout)
- Flutter 3.41.6 / Dart 3.11.4
- Local SQLite via Drift (schema v6), Riverpod for state

## Architecture

### Adapter Pattern (`lib/data/adapters/`)
- `TtsAdapter` — abstract base: `synthesize(TtsRequest)`, `healthCheck()`, `getSpeakers()`
- `OpenAiCompatibleAdapter` — `POST /audio/speech` passthrough; speakers via `GET /audio/voices` then fallback `GET /speakers`
- `ChatCompletionsTtsAdapter` — `POST /chat/completions` with audio output (MiMo/Qwen style); speakers via `GET /speakers`
- `CosyVoiceAdapter` — native CosyVoice API: `POST /cosyvoice/speech` (JSON), `POST /cosyvoice/speech/upload` (multipart with ref audio); speakers via `GET /speakers`; health via `GET /health`
- `createAdapter(provider, {modelName})` factory — dispatches on `adapterType` (`openaiCompatible`, `chatCompletionsTts`, `cosyvoice`)

### Database Schema (`lib/data/database/tables.dart`) — v6
```
TtsProviders         — upstream provider configs (with `position` for manual ordering, `enabled` defaults false)
ModelBindings        — provider <-> model mappings
VoiceAssets          — named voice characters (with avatarPath, modelName)
VoiceBanks           — named character rosters; only one active at a time
VoiceBankMembers     — junction: bank <-> asset (many-to-many)
TtsJobs              — background job queue (unused UI, reserved)
QuickTtsHistories    — persistent Quick TTS log (with audioDuration)
PhaseTtsProjects     — Phase TTS projects (name, bankId, scriptText)
PhaseTtsSegments     — segments per project (text, voiceAssetId, audioPath, audioDuration)
DialogTtsProjects    — Dialog TTS projects (name, bankId)
DialogTtsLines       — lines per dialog project (text, voiceAssetId, audioPath, audioDuration)
AudioTracks          — single audio samples library (audioPath, refText, refLang, durationSec, sourceType, avatarPath)
```

### Default Data (seeded on first launch / schema upgrade)
- **3 providers**: OpenAI TTS (`openaiCompatible`), Chat Completions TTS (`chatCompletionsTts`), CosyVoice (`cosyvoice`) — all disabled by default, pre-filled with localhost URLs
- **1 voice character**: "Default Voice" (preset voice "alloy" on OpenAI TTS)
- **1 voice bank**: "Default Bank" (active, contains Default Voice)
- **1 dialog TTS project**: "Sample Dialog"
- **1 phase TTS project**: "Sample Narration" (with placeholder script)

### Riverpod Providers (`lib/providers/app_providers.dart`)
- `databaseProvider` — singleton AppDatabase
- `ttsProvidersStreamProvider` — all providers stream (ordered: enabled first, then position)
- `voiceAssetsStreamProvider` — all voice assets stream
- `voiceBanksStreamProvider` / `activeBankStreamProvider` / `bankMembersStreamProvider(bankId)`
- `quickTtsHistoryStreamProvider`
- `phaseTtsProjectsStreamProvider` / `phaseTtsSegmentsStreamProvider(projectId)`
- `dialogTtsProjectsStreamProvider` / `dialogTtsLinesStreamProvider(projectId)`
- `audioTracksStreamProvider` — all audio tracks stream
- `serverRunningProvider` — API server toggle

### Local API Server (`lib/server/api_server.dart`)
- `GET /v1/models` — lists enabled VoiceAssets
- `POST /v1/audio/speech` — looks up voice -> provider -> adapter -> returns audio bytes

## Shared Widgets (`lib/presentation/widgets/`)

### ResizableSplitPane (`resizable_split_pane.dart`)
Reusable two-pane layout with draggable divider, used across all split-layout screens.
- **Drag divider** to resize freely; divider highlights purple on hover/drag
- **Collapse fully** by dragging past threshold (80px) — visible pane fills 100% width
- **Restore from collapsed**: edge drag zone (thin strip at collapsed edge) or prominent Back button (top-left) or Escape key
- **Edge drag zone**: when collapsed, a 5px invisible drag handle appears at the collapsed edge; hovering widens it to 8px with purple highlight; dragging it transitions into split mode with the divider following the cursor
- **Back button**: elevated Material button with arrow icon (20px) and "Back" label (14px, semi-bold), 16x10 padding, shadow for visibility
- **Saved fraction**: collapsing saves the current split ratio; expanding restores it
- **Mobile-ready**: supports `startCollapsedLeft` / `startCollapsedRight` for narrow screens

### Sidebar (`sidebar.dart`)
64px icon nav, main tabs on top, Providers + Settings pinned at bottom separated by divider.

## CosyVoice Backend Integration

The CosyVoice backend (`localhost:9880`) serves **two API styles** from a single server:

| Style | Base URL | Adapter Type | Use Case |
|---|---|---|---|
| OpenAI-compatible | `http://localhost:9880/v1` | `openaiCompatible` | Drop-in for OpenAI TTS clients; voices at `/v1/audio/voices` |
| CosyVoice Native | `http://localhost:9880` | `cosyvoice` | Full CosyVoice features: zero_shot, cross_lingual, instruct modes |

**Native API endpoints:**
- `GET /health` — health check
- `GET /speakers` — list speakers (SillyTavern format)
- `GET /cosyvoice/profiles` — list local profiles
- `GET /cosyvoice/meta` — capabilities metadata
- `POST /cosyvoice/speech` — JSON synthesis (profile, mode, instruct_text, prompt_text, speed, response_format)
- `POST /cosyvoice/speech/upload` — multipart synthesis with uploaded prompt_audio file (zero_shot)

## Known Limitations / TODO

- `TtsJobs` table exists but no background job queue UI yet
- Phase TTS "Import Text" button is not implemented
- GPT-SoVITS, Qwen3 Native adapters — stubs only (`UnimplementedError`)
- Audio waveform trimmer is text-fields only (no visual waveform)
- Windows Developer Mode required for release builds (symlinks)
- `avoid_print` lint in `lib/server/api_server.dart:35` (pre-existing)
- Phase/Dialog TTS: phone layout not yet adapted (currently desktop split-panel only, but ResizableSplitPane supports collapse for future mobile adaptation)
- Quick TTS waveform is static (decorative); no real waveform data from audio
- Voice Asset microphone recording: entry point exists but capture/editor not implemented
- Voice Asset "Add to library" from Quick/Phase/Dialog TTS screens not yet wired
- Default providers use `localhost` URLs — user must update to their actual endpoints
