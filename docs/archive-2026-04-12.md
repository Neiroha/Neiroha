# Neiroha Development Archive ‚Äî 2026-04-12

## Project Overview

**Neiroha** is a Flutter-based AI audio middleware and dubbing workstation.
- Platform: Windows / Android (responsive layout)
- Flutter 3.41.6 / Dart 3.11.4
- Local SQLite via Drift (schema v8), Riverpod for state

## Architecture

### Adapter Pattern (`lib/data/adapters/`)
- `TtsAdapter` ‚Äî abstract base: `synthesize(TtsRequest)`, `healthCheck()`, `getSpeakers()`, `getModels()`
- `ModelInfo` ‚Äî data class for model query results (`id`, `name`)
- `OpenAiCompatibleAdapter` ‚Äî `POST /audio/speech`; speakers via `GET /audio/voices` then `/speakers`; models via `GET /models`
- `ChatCompletionsTtsAdapter` ‚Äî `POST /chat/completions` with audio output (MiMo V2 / Qwen style); models via `GET /models`
- `CosyVoiceAdapter` ‚Äî native CosyVoice JSON API: `POST /cosyvoice/speech` + `/cosyvoice/speech/upload`; speakers via `GET /speakers`; health via `GET /health`
- `GptSovitsAdapter` ‚Äî GPT-SoVITS api_v2.py: `POST /tts`; health via `GET /control`; no speaker list (uses ref audio files)
- `AzureTtsAdapter` ‚Äî Azure Cognitive Services REST API with SSML: `POST /cognitiveservices/v1`; voices/models via `GET /cognitiveservices/voices/list`
- `SystemTtsAdapter` ‚Äî Windows SAPI via PowerShell `System.Speech.Synthesis.SpeechSynthesizer`; no network required
- `createAdapter(provider, {modelName})` factory ‚Äî dispatches on `adapterType`

### Database Schema (`lib/data/database/tables.dart`) ‚Äî v8
```
TtsProviders         ‚Äî upstream provider configs (with `position` for manual ordering, `enabled` defaults false)
ModelBindings        ‚Äî provider <-> model mappings (auto-fetched or manually added)
VoiceAssets          ‚Äî named voice characters (with avatarPath, modelName, modelBindingId)
VoiceBanks           ‚Äî named character rosters; multiple can be active simultaneously
VoiceBankMembers     ‚Äî junction: bank <-> asset (many-to-many)
TtsJobs              ‚Äî background job queue (unused UI, reserved)
QuickTtsHistories    ‚Äî persistent Quick TTS log (with audioDuration)
PhaseTtsProjects     ‚Äî Phase TTS projects (name, bankId, scriptText)
PhaseTtsSegments     ‚Äî segments per project (text, voiceAssetId, audioPath, audioDuration)
DialogTtsProjects    ‚Äî Dialog TTS projects (name, bankId)
DialogTtsLines       ‚Äî lines per dialog project (text, voiceAssetId, audioPath, audioDuration)
AudioTracks          ‚Äî single audio samples library (audioPath, refText, refLang, durationSec, sourceType, avatarPath)
```

### Default Data (seeded on first launch / schema upgrade)
- **6 providers** (all disabled by default):
  - OpenAI TTS (`openaiCompatible`) ‚Äî `http://localhost:8880/v1`
  - MiMo V2 TTS (`chatCompletionsTts`) ‚Äî `https://api.xiaomimimo.com/v1`, model `mimo-v2-tts`
  - CosyVoice (`cosyvoice`) ‚Äî `http://127.0.0.1:9880`
  - GPT-SoVITS (`gptSovits`) ‚Äî `http://127.0.0.1:9880`
  - Azure TTS (`azureTts`) ‚Äî `https://eastus.tts.speech.microsoft.com`
  - Windows System TTS (`systemTts`) ‚Äî no URL needed
- **1 voice character**: "Default Voice" (preset voice "alloy" on OpenAI TTS)
- **1 voice bank**: "Default Bank" (active, contains Default Voice)
- **1 dialog TTS project**: "Sample Dialog"
- **1 phase TTS project**: "Sample Narration" (with placeholder script)

### Riverpod Providers (`lib/providers/app_providers.dart`)
- `databaseProvider` ‚Äî singleton AppDatabase
- `ttsProvidersStreamProvider` ‚Äî all providers stream (ordered: enabled first, then position)
- `voiceAssetsStreamProvider` ‚Äî all voice assets stream
- `voiceBanksStreamProvider` / `activeBankStreamProvider` / `bankMembersStreamProvider(bankId)`
- `quickTtsHistoryStreamProvider`
- `phaseTtsProjectsStreamProvider` / `phaseTtsSegmentsStreamProvider(projectId)`
- `dialogTtsProjectsStreamProvider` / `dialogTtsLinesStreamProvider(projectId)`
- `audioTracksStreamProvider` ‚Äî all audio tracks stream
- `serverRunningProvider` ‚Äî API server toggle

### Local API Server (`lib/server/api_server.dart`)

The API server uses **voice banks as the model abstraction**:
- `GET /v1/models` ‚Äî lists all active voice banks as models (bank name = model ID)
- `GET /v1/audio/voices` ‚Äî lists voices from all active banks with `model` = bank name
- `GET /speakers` ‚Äî SillyTavern-compatible speaker list scoped to active banks
- `POST /v1/audio/speech` ‚Äî `model` field = bank name (scopes voice lookup); falls back to global lookup
- `GET /health` ‚Äî server health check

Multiple voice banks can be active simultaneously, each appearing as a separate "model" to external clients.

## Shared Widgets (`lib/presentation/widgets/`)

### ResizableSplitPane (`resizable_split_pane.dart`)
Reusable two-pane layout with draggable divider, used across all split-layout screens.
- **Drag divider** to resize freely; divider highlights purple on hover/drag
- **Collapse fully** by dragging past threshold (80px) ‚Äî visible pane fills 100% width
- **Restore from collapsed**: edge drag zone (thin strip at collapsed edge) or prominent Back button (top-left) or Escape key
- **Edge drag zone**: when collapsed, a 5px invisible drag handle appears at the collapsed edge; hovering widens it to 8px with purple highlight; dragging it transitions into split mode with the divider following the cursor
- **Back button**: elevated Material button with arrow icon (20px) and "Back" label (14px, semi-bold), 16x10 padding, shadow for visibility
- **Saved fraction**: collapsing saves the current split ratio; expanding restores it
- **Mobile-ready**: supports `startCollapsedLeft` / `startCollapsedRight` for narrow screens

### Sidebar (`sidebar.dart`)
64px icon nav, main tabs on top, Providers + Settings pinned at bottom separated by divider.

## CosyVoice Backend Integration

The CosyVoice backend (`127.0.0.1:9880`) serves **two API styles** from a single server:

| Style | Base URL | Adapter Type | Use Case |
|---|---|---|---|
| OpenAI-compatible | `http://127.0.0.1:9880/v1` | `openaiCompatible` | Drop-in for OpenAI TTS clients |
| CosyVoice Native | `http://127.0.0.1:9880` | `cosyvoice` | Full features: zero_shot, cross_lingual, instruct modes |

**Native API:**
- `POST /cosyvoice/speech` ‚Äî JSON synthesis (profile, mode, instruct_text, prompt_text, speed, response_format)
- `POST /cosyvoice/speech/upload` ‚Äî multipart synthesis with uploaded prompt_audio
- `GET /health`, `GET /speakers`, `GET /cosyvoice/profiles`

## Changelog ‚Äî 2026-04-12

### Added
- **GPT-SoVITS adapter** (`gptSovits`) ‚Äî full implementation based on official api_v2.py: `POST /tts` synthesis with ref_audio_path, prompt_text, prompt_lang, text_lang, speed_factor, media_type; health check via `GET /control`; no speaker list (voices defined by reference audio files)
- **Azure TTS adapter** (`azureTts`) ‚Äî Microsoft Azure Cognitive Services Speech REST API with SSML synthesis; supports voice selection, prosody rate, multiple output formats (wav/mp3/opus/pcm); voice list and health check via `GET /cognitiveservices/voices/list`; free tier 500k chars/month
- **Windows System TTS adapter** (`systemTts`) ‚Äî built-in Windows SAPI via PowerShell `System.Speech.Synthesis.SpeechSynthesizer`; zero setup on Windows 10/11; voice selection, speed mapping (-10..10 SAPI rate), WAV output; `GetInstalledVoices()` for speaker list
- **`GET /v1/audio/voices`** ‚Äî OpenAI-compatible voice listing endpoint on local API server, scoped to active voice banks
- **`GET /speakers`** ‚Äî SillyTavern-compatible speaker list endpoint on local API server
- **Model list management** ‚Äî `ModelInfo` class + `getModels()` on `TtsAdapter` base; implemented for `openaiCompatible` (GET /models), `chatCompletionsTts` (GET /models), `azureTts` (voice locales from voice list)
- **Provider editor model list UI** ‚Äî "Auto Fetch" button queries provider API for available models; "Add" button for manual model entry; models displayed as removable chips; persisted in `ModelBindings` table
- **Provider duplicate button** ‚Äî copy any provider as a template (useful for same adapter type with different URL/key)
- **Multi-bank activation** ‚Äî `getActiveBanks()`, `watchActiveBanks()`, `toggleBankActive()` in database; multiple voice banks can be active simultaneously
- **Voice bank as API model** ‚Äî active voice bank names serve as `model` values in the local API; `/v1/models` returns active banks; `POST /v1/audio/speech` accepts bank name in `model` field

### Changed
- **Chat Completions TTS default seed** renamed to **MiMo V2 TTS** with base URL `https://api.xiaomimimo.com/v1` and model `mimo-v2-tts`
- **CosyVoice default seed** ‚Äî base URL changed to `http://127.0.0.1:9880`, default model name cleared (native API doesn't use model names)
- **GPT-SoVITS default seed** ‚Äî base URL `http://127.0.0.1:9880`
- **`/v1/models` endpoint** ‚Äî now returns active voice banks instead of voice assets
- **`/v1/audio/voices` endpoint** ‚Äî voices scoped to active banks, `model` field = bank name
- **`/speakers` endpoint** ‚Äî voices scoped to active banks, includes `model` field
- **`POST /v1/audio/speech`** ‚Äî `model` field now resolves as bank name (voice lookup scoped to bank members), with global fallback
- **AdapterType enum** ‚Äî added `azureTts`, `systemTts`; added `supportsModelQuery` getter; CosyVoice default model set to empty string
- **Schema version** bumped from v7 to v8
