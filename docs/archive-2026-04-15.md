# Neiroha Development Archive — 2026-04-15

## Project Overview

**Neiroha** (formerly Q-Vox-Lab) is a Flutter-based AI audio middleware and dubbing workstation.
- Platform: Windows / Android (responsive layout)
- Flutter 3.41.6 / Dart 3.11.4
- Local SQLite via Drift (schema v8), Riverpod for state

## Architecture

### Adapter Pattern (`lib/data/adapters/`)
- `TtsAdapter` — abstract base: `synthesize(TtsRequest)`, `healthCheck()`, `getSpeakers()`, `getModels()`
- `ModelInfo` — data class for model query results (`id`, `name`)
- `OpenAiCompatibleAdapter` — `POST /audio/speech`; speakers via `GET /audio/voices` then `/speakers`; models via `GET /models`
- `ChatCompletionsTtsAdapter` — `POST /chat/completions` with audio output (MiMo V2 / Qwen style); models via `GET /models`
- `CosyVoiceAdapter` — native CosyVoice JSON API: `POST /cosyvoice/speech` + `/cosyvoice/speech/upload`; speakers via `GET /speakers`; health via `GET /health`
- `GptSovitsAdapter` — GPT-SoVITS api_v2.py: `POST /tts`; health via `GET /control`; no speaker list (uses ref audio files)
- `AzureTtsAdapter` — Azure Cognitive Services REST API with SSML: `POST /cognitiveservices/v1`; voices/models via `GET /cognitiveservices/voices/list`; URL normalization accepts bare region names, cognitive endpoint variants, or full TTS endpoints
- `SystemTtsAdapter` — Windows SAPI via PowerShell `System.Speech.Synthesis.SpeechSynthesizer`; no network required
- `createAdapter(provider, {modelName})` factory — dispatches on `adapterType`

### Database Schema (`lib/data/database/tables.dart`) — v8
```
TtsProviders         — upstream provider configs (with `position` for manual ordering, `enabled` defaults false)
ModelBindings        — provider <-> model/voice mappings (auto-fetched or manually added; `supportedTaskModes='voice'` marks voice entries)
VoiceAssets          — named voice characters (with avatarPath, modelName, modelBindingId)
VoiceBanks           — named character rosters; multiple can be active simultaneously
VoiceBankMembers     — junction: bank <-> asset (many-to-many)
TtsJobs              — background job queue (unused UI, reserved)
QuickTtsHistories    — persistent Quick TTS log (with audioDuration)
PhaseTtsProjects     — Phase TTS projects (name, bankId, scriptText)
PhaseTtsSegments     — segments per project (text, voiceAssetId, audioPath, audioDuration)
DialogTtsProjects    — Dialog TTS projects (name, bankId)
DialogTtsLines       — lines per dialog project (text, voiceAssetId, audioPath, audioDuration)
AudioTracks          — single audio samples library (audioPath, refText, refLang, durationSec, sourceType, avatarPath)
```

### Default Data (seeded on first launch / schema upgrade)
- **6 providers** (all disabled by default):
  - OpenAI TTS (`openaiCompatible`) — `http://localhost:8880/v1`
  - MiMo V2 TTS (`chatCompletionsTts`) — `https://api.xiaomimimo.com/v1`, model `mimo-v2-tts`
  - CosyVoice (`cosyvoice`) — `http://127.0.0.1:9880`
  - GPT-SoVITS (`gptSovits`) — `http://127.0.0.1:9880`
  - Azure TTS (`azureTts`) — `https://eastus.tts.speech.microsoft.com`
  - Windows System TTS (`systemTts`) — no URL needed
- **1 voice character**: "Default Voice" (preset voice "alloy" on OpenAI TTS)
- **1 voice bank**: "Default Bank" (active, contains Default Voice)
- **1 dialog TTS project**: "Sample Dialog"
- **1 phase TTS project**: "Sample Narration" (with placeholder script)

### AdapterType Enum (`lib/domain/enums/adapter_type.dart`)

Each `AdapterType` value exposes the following capability getters:

| Getter | Description |
|---|---|
| `displayName` | Human-readable name shown in UI |
| `defaultModel` | Default model key for the adapter |
| `supportsModelQuery` | Whether `GET /models` can auto-fetch models (`openaiCompatible`, `chatCompletionsTts`) |
| `supportsVoiceQuery` | Whether the adapter can fetch voices from the API (`azureTts`, `systemTts`, `openaiCompatible`, `chatCompletionsTts`) |
| `hasSeparateModelAndVoice` | Whether the adapter has distinct model + voice concepts — shows two sections in provider editor (`openaiCompatible`, `chatCompletionsTts`) |
| `showDefaultModelField` | Whether the default model name field is shown in the provider editor (false for `openaiCompatible`, `chatCompletionsTts`, `azureTts`, `systemTts`) |

### Riverpod Providers (`lib/providers/app_providers.dart`)
- `databaseProvider` — singleton AppDatabase
- `apiServerProvider` — singleton ApiServer
- `ttsProvidersStreamProvider` — all providers stream (ordered: enabled first, then position)
- `voiceAssetsStreamProvider` — all voice assets stream
- `voiceBanksStreamProvider` / `activeBankStreamProvider` / `bankMembersStreamProvider(bankId)`
- `quickTtsHistoryStreamProvider`
- `phaseTtsProjectsStreamProvider` / `phaseTtsSegmentsStreamProvider(projectId)`
- `dialogTtsProjectsStreamProvider` / `dialogTtsLinesStreamProvider(projectId)`
- `audioTracksStreamProvider` — all audio tracks stream
- `serverRunningProvider` — API server toggle
- `audioPlayerProvider` — **shared** `AudioPlayer` instance across all TTS screens; screens subscribe/unsubscribe but must NOT call `player.dispose()`

### Local API Server (`lib/server/api_server.dart`)

The API server uses **voice banks as the model abstraction**:
- `GET /v1/models` — lists all active voice banks as models (bank name = model ID)
- `GET /v1/audio/voices` — lists voices from all active banks with `model` = bank name
- `GET /speakers` — SillyTavern-compatible speaker list scoped to active banks
- `POST /v1/audio/speech` — `model` field = bank name (scopes voice lookup); falls back to global lookup
- `GET /health` — server health check

Multiple voice banks can be active simultaneously, each appearing as a separate "model" to external clients.

## Shared Widgets (`lib/presentation/widgets/`)

### ResizableSplitPane (`resizable_split_pane.dart`)
Reusable two-pane layout with draggable divider, used across all split-layout screens.
- **Drag divider** to resize freely; divider highlights purple on hover/drag
- **Collapse fully** by dragging past threshold (80px) — visible pane fills 100% width
- **Restore from collapsed**: edge drag zone (thin strip at collapsed edge) or prominent Back button (top-left) or Escape key
- **Edge drag zone**: when collapsed, a 5px invisible drag handle appears at the collapsed edge; hovering widens it to 8px with purple highlight; dragging it transitions into split mode with the divider following the cursor
- **Back button**: elevated Material button with arrow icon (20px) and "Back" label (14px, semi-bold), 16x10 padding, shadow for visibility
- **Saved fraction**: collapsing saves the current split ratio; expanding restores it
- **Mobile-ready**: supports `startCollapsedLeft` / `startCollapsedRight` for narrow screens

### Sidebar (`sidebar.dart`)
64px icon nav, main tabs on top, Providers + Settings pinned at bottom separated by divider.

## CosyVoice Backend Integration

The CosyVoice backend (`127.0.0.1:9880`) serves **two API styles** from a single server:

| Style | Base URL | Adapter Type | Use Case |
|---|---|---|---|
| OpenAI-compatible | `http://127.0.0.1:9880/v1` | `openaiCompatible` | Drop-in for OpenAI TTS clients |
| CosyVoice Native | `http://127.0.0.1:9880` | `cosyvoice` | Full features: zero_shot, cross_lingual, instruct modes |

**Native API:**
- `POST /cosyvoice/speech` — JSON synthesis (profile, mode, instruct_text, prompt_text, speed, response_format)
- `POST /cosyvoice/speech/upload` — multipart synthesis with uploaded prompt_audio
- `GET /health`, `GET /speakers`, `GET /cosyvoice/profiles`

## Azure TTS URL Normalization

`AzureTtsAdapter._normalizeTtsUrl()` accepts three forms for the base URL and converts all to the canonical TTS speech endpoint:

| Input form | Example | Output |
|---|---|---|
| Already correct TTS endpoint | `https://eastus.tts.speech.microsoft.com` | unchanged |
| Cognitive management endpoint | `https://eastus.api.cognitive.microsoft.com` | `https://eastus.tts.speech.microsoft.com` |
| Bare region name | `eastus` | `https://eastus.tts.speech.microsoft.com` |

## Database File Migration

When the app launches, `_resolveDatabaseFile()` in `AppDatabase` tries the following in order:
1. `AppSupportDir/neiroha.db` (current name)
2. `AppSupportDir/q_vox_lab.db` (old name from pre-rename era)
3. Path variants where `com.neiroha.neiroha` → `com.qvoxlab.q_vox_lab`, etc.

If a legacy file is found, it is copied to `neiroha.db` before opening. This ensures existing user data survives the project rename.

## Changelog — 2026-04-15

### Added
- **Shared AudioPlayer** (`audioPlayerProvider`) — single long-lived `AudioPlayer` instance shared across Quick TTS, Phase TTS, and Dialog TTS screens. Eliminates repeated platform-channel threading warnings on Windows (audioplayers_windows WinRT thread warning fires once per session instead of once per navigation). Screens must cancel stream subscriptions in `dispose()` but must NOT call `player.dispose()`.
- **`audioPlayerProvider`** in `lib/providers/app_providers.dart` — Riverpod `Provider<AudioPlayer>` with `ref.onDispose` cleanup
- **`AdapterType.supportsVoiceQuery`** getter — indicates which adapters can auto-fetch a voice list from the upstream API (`azureTts`, `systemTts`, `openaiCompatible`, `chatCompletionsTts`)
- **`AdapterType.hasSeparateModelAndVoice`** getter — indicates adapters with distinct model + voice concepts that need two separate sections in the provider editor (`openaiCompatible`, `chatCompletionsTts`)
- **`AdapterType.showDefaultModelField`** getter — controls visibility of the default model name text field in the provider editor (false for `openaiCompatible`, `chatCompletionsTts`, `azureTts`, `systemTts`)
- **`AppDatabase.getModelEntriesForProvider()`** — returns only model bindings (`supportedTaskModes != 'voice'`) for a provider
- **`AppDatabase.getVoiceEntriesForProvider()`** — returns only voice bindings (`supportedTaskModes == 'voice'`) for a provider
- **Azure TTS URL normalization** (`AzureTtsAdapter._normalizeTtsUrl()`) — accepts bare region names (`eastus`), cognitive management endpoints (`*.api.cognitive.microsoft.com`), and full TTS endpoints; all normalized to `https://{region}.tts.speech.microsoft.com`
- **Separate model and voice sections** in Provider editor — adapters with `hasSeparateModelAndVoice=true` (OpenAI, Chat Completions) show dedicated "Models" and "Voices" chip sections; adapters without show a unified "Models" section
- **Database file migration** (`_resolveDatabaseFile` / `_legacyDatabaseCandidates`) — on first launch after project rename, existing `q_vox_lab.db` data is migrated to `neiroha.db`
- **README.md** and **README_zh.md** — full project documentation with screenshots (overview, Quick TTS, Dialog TTS, Providers screens)
- **Screenshots** in `assets/images/` — `neiroha_logo.png`, `screenshot_overview.png`, `screenshot_quick_tts.png`, `screenshot_dialog_tts.png`, `screenshot_providers.png`

### Changed
- **Quick TTS / Phase TTS / Dialog TTS screens** — migrated from per-screen `AudioPlayer` to shared `audioPlayerProvider`; screens subscribe to player state streams but no longer own or dispose the player
