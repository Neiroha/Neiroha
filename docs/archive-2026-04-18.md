# Neiroha Development Archive — 2026-04-18

## Summary

Consolidation pass: Quick TTS is now a panel embedded inside Voice Bank (auditioning a
character during editing is a one-click action); the standalone Quick TTS tab and the
Dialog TTS timeline panel were both removed.

---

## Voice Bank ⇐ Quick TTS

The character inspector column is now vertically split. Top half is a `QuickTtsPanel`
that uses the currently-selected character as the voice; bottom half is the existing
`CharacterInspector`. The divider is draggable.

### `lib/presentation/widgets/quick_tts_panel.dart` (new)

Compact Quick TTS surface tailored for embedding. Takes an optional `VoiceAsset`;
shows a per-character history (filtered by `voiceAssetId`), a waveform + inline
play/stop control per entry, and a compact generate bar at the bottom.

### `lib/presentation/widgets/resizable_split_pane.dart`

Added `VerticalResizableSplitPane` — stacked panes with a draggable horizontal
divider. Simpler than `ResizableSplitPane`: no collapse / edge-drag support. Used
for the Quick TTS over Inspector split.

### `lib/presentation/screens/voice_bank_screen.dart`

`_buildInspectorColumn()` now returns a `VerticalResizableSplitPane` with
`QuickTtsPanel` on top and `CharacterInspector` below. Both children get a
`ValueKey(asset.id)` so controllers and text fields reset when the selected
character changes.

### `lib/data/database/app_database.dart`

Added `clearQuickTtsHistoryForAsset(String assetId)` so the embedded panel's
"clear" button only wipes history for the current character, not all of it.

## Deleted: Standalone Quick TTS Tab

### `lib/presentation/screens/quick_tts_screen.dart` (removed)

### `lib/presentation/navigation/app_navigation.dart`

Removed `quickTts` from the `NavTab` enum.

### `lib/presentation/screens/app_shell.dart`

Dropped the `quick_tts_screen.dart` import and the `NavTab.quickTts` switch case.
Default landing tab changed from `NavTab.quickTts` → `NavTab.voiceBank`.

### `lib/presentation/widgets/sidebar.dart`

Removed `NavTab.quickTts` from `mainTabs`.

Note: the `quickTtsHistories` table and all related DB methods are retained —
`QuickTtsPanel` still reads/writes them.

---

## Deleted: Dialog TTS Timeline Panel

The `StoryTrackEditor` was pulled out of `dialog_tts_screen.dart` along with its
supporting plumbing. `StoryTrackEditor` itself is kept — Phase TTS still uses it.

### `lib/presentation/screens/dialog_tts_screen.dart`

Removed:
- Import of `story_track_editor.dart` (`StoryTrackEditor`, `TimelineDragButton`,
  `TimelineDropPayload`).
- `timelineClipsStreamProvider('dialog:$projectId')` watch + the seed-on-first-view
  block and `_seededProjects` set.
- `StoryTrackEditor(...)` widget between the chat view and the action bar.
- `_addLineToTimeline`, `_syncTimelineClip`, `_seedTimeline` helper methods.
- The `_syncTimelineClip(...)` call inside `_generateOne` (line audio is still
  written to `DialogTtsLine.audioPath`; no clip row is created).
- The `TimelineDragButton` inside `_ChatBubble` and the `onAddToTimeline`
  parameter / callback wiring at the call site.

---

## Architecture Notes

- `StoryTrackEditor` is now the sole entry point to the multi-lane timeline UI,
  and only Phase TTS mounts it.
- `TimelineClips` rows tagged with `projectType = 'dialog'` become orphaned after
  this change. They are harmless; no migration was done. If needed later,
  `DELETE FROM timeline_clips WHERE project_type = 'dialog'` is a safe one-liner.
- `QuickTtsPanel` and the standalone (now-removed) Quick TTS screen shared no
  code — duplication was minimal and extraction wasn't worth the churn.
