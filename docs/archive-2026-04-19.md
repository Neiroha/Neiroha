## VoxCPM2 Native Adapter

Targets the local VoxCPM2 launcher (`docs/API.md`, default
`http://127.0.0.1:8000`). The launcher exposes an OpenAI-compatible surface
plus native `/voxcpm/*` routes and a voice registry at `/voxcpm/voices`.

Three native modes, routed via a mode string stored in
`VoiceAsset.modelName` (same pattern CosyVoice already uses):

- `design` — text-only synthesis from a natural-language voice description
- `clone` — `reference_audio` file or a registered `voice_id`
- `ultimate_clone` — `prompt_text` plus prompt/reference audio

### `lib/data/adapters/voxcpm2_native_adapter.dart` (new)

`VoxCpm2NativeAdapter` picks the endpoint by whether the ref audio is a
local file vs a remote URI:

- `POST /voxcpm/speech/upload` (multipart) when a local file is present and
  the mode is `clone` / `ultimate_clone`. `clone` sends it as
  `reference_audio`; `ultimate_clone` sends it as `prompt_audio` with the
  required `prompt_text`.
- `POST /voxcpm/speech` (JSON) otherwise. Remote refs (`http(s)://`,
  `file://`, `data:`) are passed through as `reference_audio` so the server
  fetches them itself.

Discovery:

- `getModels()` → `GET /v1/models`
- `getVoices()` → `GET /voxcpm/voices`, returns `List<VoxCpm2Voice>` with
  `id`, `displayName`, `modeHint` — mirrors CosyVoice's `getProfiles()` so
  only server-registered ids can be sent as `voice_id`
- `getSpeakers()` → just the ids from `getVoices()` for adapter-interface
  compatibility
- `healthCheck()` → `GET /health`

### `lib/domain/enums/adapter_type.dart`

Added `voxcpm2Native` between `cosyvoice` and `chatCompletionsTts`:
`displayName: 'VoxCPM2 Native'`, `defaultModel: 'voxcpm2'`, the rest of the
capability flags default to the CosyVoice-style shape (no
`supportsModelQuery` / `supportsVoiceQuery`, `showDefaultModelField: true`)
— registered voices are fetched live in the character dialog, not via the
provider screen's Model/Voice cache.

### `lib/data/adapters/tts_adapter.dart`

Factory routes `voxcpm2Native` → `VoxCpm2NativeAdapter`. Dropped the stale
`qwen3Native` TODO.

### `lib/data/database/app_database.dart`

Seeded a disabled "VoxCPM2 Native" default provider at
`http://127.0.0.1:8000` with model `voxcpm2`, inserted in position 3 (so
GPT-SoVITS/Azure/System shift to 4/5/6). Schema version 9 → 10 to trigger
re-seed of the defaults for existing installs.

### `lib/presentation/screens/provider_screen.dart`

Base-URL hint for `voxcpm2Native`: `http://127.0.0.1:8000`.

### `lib/presentation/screens/voice_character_screen.dart`

`CreateCharacterDialog` now has a dedicated `voxcpm2Native` branch:

- New `_VoxCpm2ModeSelector` (Design / Clone / Ultra Clone), parallel to
  `_CosyVoiceModeSelector`.
- `_fetchVoxCpm2Voices()` live-fetches registered voices when the provider
  is VoxCPM2, using the adapter's `getVoices()`.
- Registered-voice dropdown (clone / ultimate_clone only) writes its
  selection into `_voiceNameCtrl`, which is persisted as
  `presetVoiceName` and sent as `voice_id` at synthesis time.
- Mode-specific form:
  - Design → `_instructionCtrl` holds the natural-language voice description
    (required).
  - Clone → reference-audio picker (required unless a registered voice is
    chosen).
  - Ultra Clone → reference-audio picker + `_promptTextCtrl` (both required).
- On save, the mode is written into `VoiceAsset.modelName` — identical to
  the CosyVoice path — so `createAdapter(provider, modelName: asset.modelName)`
  routes correctly at synth time.
- `_save()` picks up dedicated validation messages for each VoxCPM2 mode.

## Quick TTS UX

### `lib/presentation/widgets/quick_tts_panel.dart`

Removed the fake sine-wave visualization (`List.generate(24, …)` of bars
derived from `math.sin(...)`) — it was decorative and competed with the
real playback UI provided by `PersistentAudioBar`. The `dart:math` import
was dropped.

The audio row is now: play button · status text ("Tap to play" / "Playing…")
· duration. Progress and scrubbing remain in the persistent bar below.

New `_saveEntryAsVoiceAsset(entry)` triggered by a "Save as Voice Asset"
button on each history row:

- Copies the Quick-TTS wav from `<appSupport>/quick_tts/` into
  `<appSupport>/voice_assets/<uuid>.<ext>`.
- Inserts an `AudioTrack` with `sourceType: 'quickTts'`,
  `refText = entry.inputText`, `refLang = asset.promptLang`,
  `durationSec` forwarded from the entry, `name =
  "{voice} · {input preview}"` (input trimmed to 24 chars).
- Shows a SnackBar confirmation.

Intended use case: VoxCPM2 **Design** mode. Iterate on a voice prompt in
Quick TTS, save the take you like as an asset, then reuse that audio as
`reference_audio` for a Clone / Ultra Clone character.

## Bug Fix — Voice Bank dispose crash

Two related crashes when navigating away from the Voice Bank tab:

1. `Bad state: Cannot use "ref" after the widget was disposed.` — `dispose()`
   called `ref.read(...)`, but the `ConsumerStatefulElement` has already
   been disposed by then.
2. `Tried to modify a provider while the widget tree was building.` —
   writing to a `StateProvider` during the unmount pass of
   `BuildOwner.finalizeTree`.

### `lib/presentation/screens/voice_bank_screen.dart`

- Cache `selectedCharacterIdProvider.notifier` and
  `playbackNotifierProvider.notifier` in `initState` (`_selectedCharacterCtl`,
  `_playbackNotifier`). The notifiers live in the enclosing ProviderScope
  so the references stay valid after the widget element is torn down.
- In `dispose()`, wrap the provider mutations in `Future(() { … })` so the
  `state = null` and `stopIfSourceTag(...)` calls run on the next
  event-loop tick, after `finalizeTree` releases Riverpod's
  "no mutations while the tree is building" guard.
- `_clearVoiceBankInspectorState()` kept for in-app callers (button
  handlers) that run while the widget is still mounted — those don't need
  the defer.
