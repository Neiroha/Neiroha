# Neiroha Development Archive — 2026-04-06

## Project Overview

**Neiroha** is a Flutter-based AI audio middleware and dubbing workstation.
- Platform: Windows / Android (responsive layout)
- Flutter 3.41.6 / Dart 3.11.4
- Local SQLite via Drift (schema v5), Riverpod for state

## Architecture

### Adapter Pattern (`lib/data/adapters/`)
- `TtsAdapter` — abstract base: `synthesize(TtsRequest)`, `healthCheck()`, `getSpeakers()`
- `OpenAiCompatibleAdapter` — `POST /audio/speech` passthrough; speakers via `GET /audio/voices` then fallback `GET /speakers`
- `ChatCompletionsTtsAdapter` — `POST /chat/completions` with audio output (MiMo/Qwen style); speakers via `GET /speakers`
- `CosyVoiceAdapter` — native CosyVoice API: `POST /cosyvoice/speech` (JSON), `POST /cosyvoice/speech/upload` (multipart with ref audio); speakers via `GET /speakers`; health via `GET /health`
- `createAdapter(provider, {modelName})` factory — dispatches on `adapterType` (`openaiCompatible`, `chatCompletionsTts`, `cosyvoice`)

### Database Schema (`lib/data/database/tables.dart`) — v5
```
TtsProviders         — upstream provider configs
ModelBindings        — provider ↔ model mappings
VoiceAssets          — named voice characters (with avatarPath, modelName)
VoiceBanks           — named character rosters; only one active at a time
VoiceBankMembers     — junction: bank ↔ asset (many-to-many)
TtsJobs              — background job queue (unused UI, reserved)
QuickTtsHistories    — persistent Quick TTS log (with audioDuration)
PhaseTtsProjects     — Phase TTS projects (name, bankId, scriptText)
PhaseTtsSegments     — segments per project (text, voiceAssetId, audioPath, audioDuration)
DialogTtsProjects    — Dialog TTS projects (name, bankId)
DialogTtsLines       — lines per dialog project (text, voiceAssetId, audioPath, audioDuration)
```

### Riverpod Providers (`lib/providers/app_providers.dart`)
- `databaseProvider` — singleton AppDatabase
- `ttsProvidersStreamProvider` — all providers stream
- `voiceAssetsStreamProvider` — all voice assets stream
- `voiceBanksStreamProvider` / `activeBankStreamProvider` / `bankMembersStreamProvider(bankId)`
- `quickTtsHistoryStreamProvider`
- `phaseTtsProjectsStreamProvider` / `phaseTtsSegmentsStreamProvider(projectId)`
- `dialogTtsProjectsStreamProvider` / `dialogTtsLinesStreamProvider(projectId)`
- `serverRunningProvider` — API server toggle

### Local API Server (`lib/server/api_server.dart`)
- `GET /v1/models` — lists enabled VoiceAssets
- `POST /v1/audio/speech` — looks up voice → provider → adapter → returns audio bytes

## Screens (`lib/presentation/screens/`)

### Quick TTS (`quick_tts_screen.dart`)
- Left: scrollable voice list with avatar, name, mode label (from active bank)
- Right: generation history — 3-line card layout per entry:
  - Line 1: character avatar + name + delete button (per-item)
  - Line 2: input text (2-line ellipsis)
  - Line 3: play/stop button + waveform bars (30% width, 20 bars) + mm:ss duration
- Header: "Delete All" button with 2-click confirmation (outline → red confirm)
- Bottom: text input + generate button
- Auto-plays after generation; detects and stores `audioDuration` via `AudioPlayer.getDuration()`

### Phase TTS (`phase_tts_screen.dart`)
- Left: project list (SQLite), create/delete projects; each project bound to a Voice Bank
- Right: split view — script editor (auto-saved) + segment panel
- Auto Split: splits script by double newlines → creates `PhaseTtsSegments` in DB
- Each segment has independent voice dropdown (from project's bank)
- Generate All: synthesizes each segment with its assigned voice, saves to `AppSupportDir/phase_tts/{projectId}/`

### Dialog TTS (`dialog_tts_screen.dart`)
- Left: project list (SQLite), each project bound to a Voice Bank
- Right: Telegram-like chat view with character avatar, name header, text bubble, waveform bars (40 bars, full width), play/stop button, mm:ss duration
- Input bar: voice picker + text field + send → appends `DialogTtsLine` to DB
- Generate All: batch synthesizes ungenerated lines, detects duration, saves to `AppSupportDir/dialog_tts/{projectId}/`

### Voice Characters (`voice_character_screen.dart`)
- List + inspector split layout
- Create dialog — provider-aware UI:
  - **Speaker-only providers** (`chatCompletionsTts`, `openaiCompatible`): no mode selector, forced `presetVoice`, only speaker dropdown + voice name field
  - **CosyVoice** (`cosyvoice`): 3-mode selector (zero_shot / cross_lingual / instruct), mode-specific fields:
    - All modes: speaker/profile picker
    - zero_shot: + ref audio picker + prompt text + language code
    - instruct: + instruct text field
    - cross_lingual: speaker only
  - **Other providers**: generic 3-option mode selector (Preset Voice / Voice Clone / Voice Design) with full field sets
  - **Avatar picker** in create dialog: circular avatar with camera icon overlay, picks image before saving
- Inspector: avatar (tapable → file picker), all fields display, enabled toggle
- Edit / Duplicate / Delete actions

### Voice Bank (`voice_bank_screen.dart`)
- Left: bank list — create, activate/deactivate (only one active at a time), duplicate (copies all members), delete
- Right: roster for selected bank — members from `VoiceBankMembers`
- Add Character dialog: search by name, add single (closes) or Add All
- Health Check: calls `adapter.healthCheck()` per member's provider, shows results dialog

### Providers (`provider_screen.dart`)
- Add/edit/delete TTS providers
- Adapter type dropdown: `OpenAI TTS API Compatible`, `OpenAI Chat API Completion`, `CosyVoice`
- Dialog uses `Dialog` + `Flexible` scroll to prevent overflow on small windows

### Provider Test (`provider_test_screen.dart`)
- Test endpoint health, synthesize sample text

## CosyVoice Backend Integration

The CosyVoice backend (`localhost:9880`) serves **two API styles** from a single server:

| Style | Base URL | Adapter Type | Use Case |
|---|---|---|---|
| OpenAI-compatible | `http://localhost:9880/v1` | `openaiCompatible` | Drop-in for OpenAI TTS clients; voices at `/v1/audio/voices` |
| CosyVoice Native | `http://localhost:9880` | `cosyvoice` | Full CosyVoice features: zero_shot, cross_lingual, instruct modes |

**Native API endpoints:**
- `GET /health` — health check
- `GET /speakers` — list speakers (SillyTavern format)
- `GET /cosyvoice/profiles` — list local profiles
- `GET /cosyvoice/meta` — capabilities metadata
- `POST /cosyvoice/speech` — JSON synthesis (profile, mode, instruct_text, prompt_text, speed, response_format)
- `POST /cosyvoice/speech/upload` — multipart synthesis with uploaded prompt_audio file (zero_shot)

**NativeSpeechRequest fields:** `text` (required), `mode` (zero_shot|cross_lingual|instruct), `profile`, `character_name`, `prompt_audio_path`, `prompt_text`, `instruct_text`, `prompt_lang`, `speed` (default 1.0), `response_format` (default wav)
