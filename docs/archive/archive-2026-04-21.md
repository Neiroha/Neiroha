# Neiroha Development Archive — 2026-04-21

## Summary

Reorganised on-disk storage around a portable `./data` + configurable
`./voice_asset` layout, introduced a startup scan that flags missing audio
rows without deleting them, added a settings panel for choosing the voice
asset root and clearing all archived audio behind a double confirmation,
and hardened path naming against cross-platform filesystem pitfalls.
Folder names are now pinned via a `folderSlug` column so renaming a
character or project no longer rehomes its archive. Filenames switched
from `millisecondsSinceEpoch` to the human-readable
`2026-04-21_14-30-22` format with collision dedupe.

## New storage layout

```
{appRoot}/
  data/
    neiroha.db        # SQLite, non-configurable location
    avatars/          # character + track avatar images
  voice_asset/        # default; overridable from Settings
    quick_tts/{slug}/
    phase_tts/{slug}/
    dialog_tts/{slug}/
    voice_character_ref/
    {project}/sfx/    # per-project imported SFX
```

`{appRoot}` is the executable directory when writable (portable mode),
falling back to `getApplicationSupportDirectory()` otherwise — detected
via a write-probe at startup. `{voiceAssetRoot}` defaults to
`{appRoot}/voice_asset` but users can point it anywhere from
Settings → Storage.

### `lib/data/storage/path_service.dart` (new)

Single resolver for every on-disk path. Initialised once before DB open.

- `init()` — resolves app root, creates `data/` + default
  `voice_asset/`, runs the write-probe for portable detection.
- `applyVoiceAssetRootOverride(path?)` — live switch for user-chosen
  roots. Caller is responsible for persisting.
- `sanitizeSegment(input, {fallback})` — strips reserved chars + control
  chars, collapses whitespace, strips trailing dots/spaces (both break on
  Windows), caps at 48 chars (headroom under MAX_PATH=260), appends `_`
  when the result matches a Windows reserved device name
  (`CON`/`PRN`/`AUX`/`NUL`/`COM1-9`/`LPT1-9`).
- `formatTimestamp([dt])` — returns `2026-04-21_14-30-22` (zero-padded,
  lexicographically sortable).
- `dedupeFilename(dir, base, ext)` — appends `-1`, `-2`, … on collision
  so two files with the same display name never clobber each other.
- `quickTtsDir(slug)` / `phaseTtsDir(slug)` / `dialogTtsDir(slug)` /
  `voiceCharacterRefDir()` / `avatarsDir()` / `timelineSfxDir(...)` — all
  `_ensure(...)` the folder on the way out.

`sanitizeSegment` is idempotent, so passing a pre-resolved slug through
these helpers is a no-op.

### `lib/data/storage/storage_service.dart` (new)

Orchestrates disk ↔ SQLite consistency.

- `applyPersistedRoot()` — reads `AppSettings['voiceAssetRoot']` and
  applies it to `PathService`. Called from `storageStartupProvider`.
- `setVoiceAssetRoot(path?)` — persists the override (or clears it) and
  live-applies. Pre-creates the directory so the caller gets immediate
  feedback on invalid paths.
- `scan()` — walks every audio-bearing table (`QuickTtsHistories`,
  `PhaseTtsSegments`, `DialogTtsLines`, `AudioTracks`, `TimelineClips`)
  and flips the `missing` flag to match disk. Never deletes rows — a
  gone file might mean the user moved the root, and the UI should prompt
  rather than destroy. Returns `StorageScanReport(checked, missing,
  recovered)`.
- `measureVoiceAssetRoot()` — aggregate footprint for the clear-all
  confirmation dialog.
- `clearAllAudioArchives()` — removes only the four managed subfolders
  under the voice-asset root (`quick_tts`, `phase_tts`, `dialog_tts`,
  `voice_character_ref`), then wipes audio rows in SQLite. Project
  scripts, character rows, providers and banks are preserved. Nulls
  `refAudioPath` on voice characters so the column doesn't dangle.
- `ensureVoiceAssetSlug(id)` / `ensurePhaseProjectSlug(id)` /
  `ensureDialogProjectSlug(id)` — resolve the entity's `folderSlug`,
  creating it from the sanitised display name on first call and
  persisting it. Dedupes against other rows' slugs in the same table.
  Once set, never rewritten — so renaming the display name doesn't
  rehome the archive folder.

## Schema changes (v12)

### `lib/data/database/tables.dart`

- New `AppSettings` table (`key` PK, `value`) — app-wide key/value
  store. First consumer: `voiceAssetRoot`.
- `missing` bool column on `AudioTracks`, `QuickTtsHistories`,
  `PhaseTtsSegments`, `DialogTtsLines`, `TimelineClips` (all default
  `false`) — set by the startup scan.
- `folderSlug` nullable text column on `VoiceAssets`,
  `PhaseTtsProjects`, `DialogTtsProjects` — pinned on first generation.

Schema version bumped to **12** with the existing drop-and-recreate
migration pattern.

### `lib/data/database/app_database.dart`

New DAO methods:

- `getSetting(key)` / `setSetting(key, value)` / `deleteSetting(key)`
- `markQuickTtsMissing(id, bool)` and four analogues on the other audio
  tables
- `getAllQuickTtsHistoryRaw()` and four analogues — used by the scan
- `getAllPhaseTtsProjects()` / `getPhaseTtsProjectById(id)`,
  `getAllDialogTtsProjects()` / `getDialogTtsProjectById(id)`,
  `getVoiceAssetById(id)` — used by the ensure-slug helpers
- `clearAllAudioArchives()` — transaction that deletes audio-only rows
  and nulls audio columns on project-script rows

`_openConnection()` resolves the DB file from `PathService.dataRoot`,
copying a legacy `getApplicationSupportDirectory()` copy forward on
first run so existing installs migrate cleanly.

## Providers

### `lib/providers/app_providers.dart`

- `storageServiceProvider` — wraps the DB with the storage orchestrator.
- `storageStartupProvider` — `FutureProvider<StorageScanReport>` that
  runs `applyPersistedRoot()` + `scan()` once per session. The Settings
  screen watches this and shows a warning badge when any files are
  missing. `ref.invalidate(storageStartupProvider)` re-runs it after the
  user changes the root or runs a manual sync.

### `lib/main.dart`

Calls `await PathService.instance.init()` before `runApp`, so both the
LazyDatabase opener and the first render see a consistent root.

## Settings → Storage panel

### `lib/presentation/screens/settings_screen.dart`

Rewritten around a new `_StorageCard`:

- **Data Directory** — shows the resolved `data/` path, indicates
  Portable vs App-support fallback, offers a copy-to-clipboard button.
- **Voice Asset Directory** — shows the active path plus
  Default/Custom indicator. `Change` opens a native directory picker
  and calls `StorageService.setVoiceAssetRoot(picked)`. `Reset` reverts
  to default (only shown when custom).
- **Sync with Disk** — shows scan result (`All N file(s) are present` /
  `N archived file(s) are missing on disk — rows flagged, not deleted`).
  Loading state shows a spinner; `Scan Now` invalidates the provider.
- **Clear All Archived Audio** — red-accented `Clear…` button opens the
  `_ClearAudioDialog`. The dialog shows measured file count and total
  bytes, then requires both a checkbox acknowledgement AND typing
  `CLEAR` into a confirmation field before the primary action enables.

## Call-site migrations

Every previously `getApplicationSupportDirectory()`-driven path now
routes through `PathService`:

| Site | Before | After |
|---|---|---|
| `quick_tts_panel.dart` — generation | `{appSupport}/tts_out/{name}/{epoch}.{ext}` | `{voiceAssetRoot}/quick_tts/{slug}/{timestamp}.{ext}` |
| `quick_tts_panel.dart` — save-as-asset | `{appSupport}/tts_out/{uuid}{ext}` | `{voiceAssetRoot}/voice_character_ref/{voice}_{text24}_{timestamp}{ext}` |
| `phase_tts_screen.dart` — generation | `{appSupport}/phase_tts/{name}/seg_{i}_{epoch}.{ext}` | `{voiceAssetRoot}/phase_tts/{slug}/seg_{i}_{timestamp}.{ext}` |
| `dialog_tts_screen.dart` — generation | `{appSupport}/dialog_tts/{name}/line_{i}_{epoch}.{ext}` | `{voiceAssetRoot}/dialog_tts/{slug}/line_{i}_{timestamp}.{ext}` |
| `story_track_editor.dart` — SFX import | `{appSupport}/.../sfx_{epoch}{ext}` | `{voiceAssetRoot}/{projectType}_tts/{slug}/sfx/sfx_{timestamp}{ext}` |
| `voice_asset_screen.dart` — upload | `{appSupport}/voice_assets/{uuid}{ext}` | `{voiceAssetRoot}/voice_character_ref/{basename}_{timestamp}{ext}` |
| `voice_character_screen.dart` — ref upload | `{appSupport}/voice_assets/{uuid}{ext}` | `{voiceAssetRoot}/voice_character_ref/{basename}_{timestamp}{ext}` |
| `voice_character_screen.dart` — avatar | `{appSupport}/avatars/{id}{ext}` | `{appRoot}/data/avatars/{id}{ext}` |

All generation sites use `ensure*Slug()` to resolve the stable folder
slug before writing, and `dedupeFilename()` to avoid clobbering on
timestamp collisions (sub-second regeneration).

`StoryTrackEditor` gained a `projectName` parameter from the Phase
screen so it can request `timelineSfxDir(...)`.

## Cross-platform hardening

The new sanitiser addresses three concrete Windows hazards:

1. **Reserved device names** — creating `CON/`, `PRN/`, or `COM1.wav`
   silently fails or opens the device. Any match (case-insensitive) gets
   a `_` suffix.
2. **Trailing dots / spaces** — Windows quietly strips both, producing
   DB-vs-disk drift where the DB stores `"Alice "` but disk holds
   `"Alice"`. Stripped during sanitisation.
3. **Path length** — segments capped at 48 chars. With `MAX_PATH=260`
   on Windows, three 48-char segments + timestamp + extension still
   leaves ~80 chars of headroom for the root path.

Plus the obvious Unix/Windows-invalid characters
(`<>:"/\|?*` and `\x00-\x1F`) are replaced with `_`.
