## SRT / LRC parsing

### `lib/data/storage/subtitle_parser.dart` (new)

`SubtitleParser.parseFile(File)` auto-detects format from the extension
(`.lrc` → LRC, else SRT) and returns a `List<ParsedCue>` of
`{ startMs, endMs, text }`.

- `_decodeText(bytes)` — skips a UTF-8 BOM, falls back to `latin1` with
  `allowInvalid: true` if the file isn't valid UTF-8 (common for legacy
  SRT packs from Asian subtitle sites).
- `parseSrt(text)` — splits on blank-line boundaries, finds the first
  line in each block that contains `-->`, parses the
  `HH:MM:SS,mmm --> HH:MM:SS,mmm` timecode (both `,` and `.` accepted as
  the millisecond separator), joins the remaining lines as cue text, and
  strips `<...>` / `{...}` styling tags so they don't end up read aloud.
- `parseLrc(text)` — scans for `[mm:ss.xx]` / `[mm:ss:xx]` / `[mm:ss]`
  timestamps, expands multi-timestamp lines, skips `[ar:]` / `[ti:]`
  metadata headers, sorts by start, and derives each cue's `endMs` from
  the next cue's start (last cue gets a 3-second fallback — LRC has no
  native end time).

Zero new dependencies — the parser is a couple of regexes plus
`dart:convert`.

## Cue-synced timeline widget

### `lib/presentation/widgets/video_dub_timeline.dart` (new)

`VideoDubTimeline` is purpose-built for Video Dub and deliberately
separate from `StoryTrackEditor` (which models `TimelineClips` with
free-form drag and multi-lane layout). Video-dub cues are fixed to
absolute time windows driven by the subtitle import, so a simpler,
single-lane, position-driven widget was the right shape.

- Inputs: `cues`, `assetMap` (for colouring), `position`, `duration`,
  `selectedCueId`, `onSeek(Duration)`, `onTapCue(SubtitleCue)`.
- Ruler at the top; tap-to-seek. Pixel-per-second zoom is local state
  (±, clamped 5..200).
- Cue blocks render at `startMs..endMs`, coloured by status —
  red-accented for `error`, accent for `hasAudio`, muted grey for
  not-yet-generated. Selected cue gets an amber border.
- Draggable playhead at the top. Drag updates a local `_draggingMs`
  (committed to `onSeek` on release); while not dragging it tracks
  `widget.position` so it follows the video.

Intentionally **not** wired to the `TimelineClips` table — subtitle cues
already carry their own start/end/audio, so an extra mirror row would
just drift out of sync.

## Video Dub editor rewrite

### `lib/presentation/screens/video_dub_screen.dart`

Listing mode unchanged. Editor (`_VideoDubEditor`) was rewritten around
the new parser + timeline + a dedicated cue audio player.

**Audio / video**

- Existing `media_kit` `Player` for video. Added a separate
  `audioplayers.AudioPlayer _cuePlayer` owned by the editor state — the
  shared `audioPlayerProvider` is reserved for UI-level playback (e.g.
  Voice Bank preview) and shouldn't be hijacked by the video position
  stream.
- `_onVideoTick(Duration)` runs on every `_player.stream.position`
  event. It finds the cue whose `[startMs, endMs)` window contains the
  current ms, and only acts when the active cue **changes** — stops any
  current dub audio, then (if the new cue has `audioPath`) starts it at
  `ms - cue.startMs` so scrubbing into the middle of a cue doesn't
  replay it from zero.
- Video pause/resume mirrors onto the dub player. Seeks stop the dub
  player and clear `_activeCueId` so the next tick re-evaluates.
- Transport toolbar gained two toggles:
  - **Mute original** — default *on*, since dubbing. Implemented via
    `player.setVolume(0|100)`.
  - **Sync dub** — default *on*. Flipping it off releases the cue
    player immediately.

**Subtitle panel**

- **Import SRT/LRC** — `FilePicker.platform.pickFiles(type: custom,
  allowedExtensions: [srt, lrc, vtt, txt])`, runs `SubtitleParser`,
  confirms `Append | Replace | Cancel` when the project already has
  cues, then bulk-inserts. Every imported cue is pre-assigned the first
  voice in the project's bank (SRT has no speaker field; LRC speaker
  tagging is deferred — see follow-ups).
- **Add cue** — `_addCueDialog` pre-fills `start = current playhead`,
  `end = start + 3s`. Shared dialog helper `_showCueEditDialog` handles
  both add and edit. Timestamp input accepts `mm:ss`, `mm:ss.ms`,
  `HH:mm:ss`, `HH:mm:ss.ms`; invalid input shows an inline error and
  keeps the dialog open.
- **Edit cue** — same dialog, prefilled from the row. Edits **null out
  `audioPath` + `audioDuration` + `error`** so stale audio doesn't sit
  under new text/timing; the next generation call writes fresh audio.
- **Delete cue** and **Clear all cues** (with confirm) — disk files are
  retained; only DB references are dropped. The startup storage scan
  will flag any orphans.
- Each `_CueCard` has: index badge, `mm:ss.ms → mm:ss.ms` timestamps,
  status icon (generating / error / play / pending), generate /
  regenerate, edit, delete, and a voice dropdown keyed to the project
  bank's members. Tapping a card selects it + seeks the video.

**Generation**

- `_generateOne` mirrors the Phase / Dialog TTS shape: resolve the
  asset + provider, build a `TtsRequest`, call the adapter, write to
  `{voiceAssetRoot}/video_dub/{slug}/cue_{orderIndex}_{timestamp}.{ext}`
  (dedupe-suffixed), probe the duration, persist to the cue row. Errors
  land in `cue.error` with the stale audio path cleared.
- `_generateAll` iterates cues, skipping any that already have audio or
  lack a voice assignment.

## Follow-ups / open items

- **LRC speaker tags** — `[speaker] lyric` isn't parsed yet. All
  imported cues get the bank's first voice and the user reassigns per
  cue. Auto-mapping `[speaker]` → a voice would close this.
- **Subtitle export / burn-in** — still scoped out.
- **Overlap handling** — two cues whose windows overlap both count as
  "active"; `_onVideoTick` picks the first match. No UI warning yet —
  user resolves by editing cue times.
- **Offset ducking** — if the cue's TTS audio is longer than the cue
  window, it keeps playing past `endMs` until either the next cue
  starts (which stops it) or the video seeks. No automatic fade or
  truncation.

## Part 2 (same day) — Multi-track timeline + FFmpeg integration

A second pass on the Video Dub editor. The placeholder cue strip was
replaced with a proper multi-track timeline (V1 / A1 TTS / A2, plus
optional V2 / A3), conventional zoom buttons were retired in favour of
a Premiere-style dual-handle range scrubber, and media import
(video / image / audio) was wired onto `TimelineClips`. A new FFmpeg
service powers waveform extraction; the executable is resolved from a
new Settings card (auto-detect from PATH or explicit override) so the
app stays ffmpeg-agnostic and we avoid LGPL/GPL bundling problems.

### `lib/data/storage/ffmpeg_service.dart` (new)

System-ffmpeg resolver, never bundled.

- `resolvePath()` — returns the user's override from
  `AppSettings['ffmpegPath']` first, then auto-detects via
  `where` (Windows) / `which` (POSIX), falling back to the literal
  `ffmpeg` so `Process.run` can still succeed if the current PATH
  already sees it.
- `isAvailable()` — probes `ffmpeg -version`; cached until
  `invalidate()` is called (or the override is reset).
- `extractWaveformPeaks(mediaPath, {bucketCount, sampleRate})` —
  shells out to
  `ffmpeg -v error -i <path> -ac 1 -ar 8000 -f s16le -`, decodes the
  raw PCM16 LE bytes, reduces to `bucketCount` normalised peaks
  (abs-max within each window). Returns `null` on failure so callers
  degrade gracefully.

### `lib/presentation/screens/settings_screen.dart`

New `_FfmpegCard` under a fresh **MEDIA TOOLS** section:

- Status row — `isAvailable` probe result, "Detected" vs "Not found"
  with guidance, plus a **Re-check** button that invalidates both the
  service cache and the `ffmpegAvailabilityProvider`.
- Path row — text field with a folder-open **Browse…** button; Save
  persists, Reset clears the override and reverts to PATH lookup.

### `lib/providers/app_providers.dart`

- `ffmpegServiceProvider` — singleton `FFmpegService`.
- `ffmpegAvailabilityProvider` — `FutureProvider<bool>` that the
  Settings card and the Video Dub editor both watch; invalidated
  after the user edits the override.

### `lib/presentation/widgets/video_dub_timeline.dart` (rewritten)

Single-lane cue strip → proper multi-track editor.

- **Track model** — five fixed slots:
  - **V2** `laneIndex = -2` (hidden by default)
  - **V1** `laneIndex = -1` — imported videos + images stack here
  - **A1** — TTS cues, synthesised from `SubtitleCues` (not stored in
    `TimelineClips`). Single source of truth for spoken content.
  - **A2** `laneIndex =  1` — imported audio (.wav/.mp3/...)
  - **A3** `laneIndex =  2` (hidden by default)

  Track kind encoded via the existing `sourceType` column:
  `'video'` / `'image'` for video lanes, `'imported'` for audio. No
  schema migration — avoids a forced DB drop for existing installs.

- **Track headers** — label + visibility-off button on the removable
  rows. Toolbar **Tracks** menu adds/removes V2 and A3.

- **Import menu** — toolbar popup with three targets (Video → V1,
  Image → V1, Audio → A2). The callback (`onImport(DubImportKind)`)
  keeps the widget pure; the editor does the file-copy and DB insert.

- **Range scrubber** at the bottom — Premiere-style. Two draggable
  handles on a mini overview strip (shows all clips + cues in
  miniature, plus the playhead position as an amber tick). Dragging
  the handles together zooms the view into that range; spreading them
  apart zooms out. Tapping inside the band seeks. A 500 ms minimum
  span stops the math from collapsing.

- **Waveform strip** (`plan.md` #2 landed) — when the editor supplies
  waveform peaks, they render as a centered bar-graph behind the **A1**
  cue lane so the user can align cue edges to speech boundaries.
  Peaks are normalised [0, 1]; rendering uses a one-pixel column per
  x with amplitude scaled to ~45% of the lane height.

- **Banner** — if `ffmpegAvailable == false`, a thin amber banner
  between the toolbar and track area tells the user waveforms are
  skipped; the **Settings** link flips `selectedTabProvider` to
  `NavTab.settings`.

- Helpers exposed alongside the widget: `DubLanes` constants,
  `DubImportKind` enum, `laneAndSourceForImport(kind)` and
  `makeDubClipCompanion(...)` so the editor doesn't re-import drift's
  `Value` or re-derive the encoding.

### `lib/presentation/screens/video_dub_screen.dart`

- Watches `timelineClipsStreamProvider('videodub:${projectId}')` and
  `ffmpegAvailabilityProvider`, feeds them into `VideoDubTimeline`.
- Selected-clip state (`_selectedClipId`) mirrors `_selectedCueId`
  with mutual exclusion so the amber selection border is unambiguous.
- `_refreshWaveform(path)` runs on every new video path, cached per
  path so switching back doesn't reprobe. Falls back silently if
  ffmpeg is unavailable or the extract fails.
- `_importMedia(project, kind, existing)` handles the import drop:
  file-copy into `{voiceAssetRoot}/video_dub/{slug}/assets/`
  (dedupe-named), duration probe via `measureAudioDuration` for
  audio, 3-second default for images, null for videos (awaiting a
  future ffprobe wire-up). The clip is appended after the last clip
  on the same lane (stack semantics).
- `onTapClip` seeks the video and selects the clip; `onDeleteClip`
  removes the row (the on-disk file is kept — the startup storage
  scan will flag it if needed).

### Encoding decision: no schema change

`TimelineClips` already carries enough columns (`projectId`,
`projectType`, `laneIndex`, `startTimeMs`, `durationSec`, `audioPath`,
`sourceType`, `label`) for the multi-track model. Video-dub clips use
`projectType = 'videodub'` (independent from existing phase/dialog
queries), `laneIndex` encodes the lane slot, `sourceType` encodes the
media kind. The `audioPath` column name is slightly misleading for
video/image clips but it's a pragmatic reuse — renaming would be a
disruptive migration for no user-visible benefit.

### Cross-cutting

- `storage_service.dart` — `clearAllAudioArchives` already sweeps
  `video_dub/`; no change needed here.
- `pubspec.yaml` — no new dependencies. `ffmpeg` is the user's own
  binary; waveform decoding is pure Dart.
