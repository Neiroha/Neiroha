`docs/` was excluded from git; every new archive had to be force-added.

**Files**: `.gitignore`.

## Waveform extraction: streaming instead of buffering

`FFmpegService.extractWaveformPeaks` used `Process.run`, which buffers
the entire stdout. At 8 kHz mono PCM16 that's ~115 MB for a 2-hour
video, ~230 MB for 4 hours — real OOM risk on any production-length
input.

Rewritten to stream via `Process.start`:

- **`probeDurationSeconds(mediaPath)`** (new, public) — shells out to
  `ffprobe -show_entries format=duration` so we know the sample count
  up front.
- **`PeakReducer`** (new, `@visibleForTesting`) — stateful reducer that
  accepts arbitrary PCM byte chunks via `addBytes`, handles odd-byte
  boundaries via a `_pendingLow` carry, and emits one peak per bucket
  via `finish()`. Bucket boundaries are precomputed once from the
  expected sample count so each sample hits O(1) work.
- **`extractWaveformPeaks`** now calls `Process.start`, pipes stderr
  into a `drain<void>()` (so a full stderr pipe can't stall the
  decode), and feeds stdout chunks directly to the reducer. Memory
  stays flat in the low-KB range regardless of video length.

The legacy whole-buffer path is preserved as `FFmpegService.reduceToPeaks`
(public, `@visibleForTesting`) so a future
`test/ffmpeg_peaks_test.dart` has a synchronous entry point.

**Files**: `lib/data/storage/ffmpeg_service.dart`.

## `_onVideoTick` reentrancy guard

The media_kit position stream fires ~30×/s and the tick body is async
(`stop → play → seek` on cue boundaries). Overlapping ticks caused
seeks from tick N+1 to land on tick N's stale play call — manifested
as cues replaying from zero or going silent.

Fix is a one-flight bool: `_ticking` guards the body with an early
return; a `try/finally` releases the guard.

**Files**: `lib/presentation/screens/video_dub_screen.dart`.

## Ruler tap uses live `pxPerMs`, not stale `context.size`

`_pxToMs(px)` called `context.size?.width`, which is `null` until the
first layout pass completes. The first ruler tap after opening an
editor computed body width as `0 - _headerWidth`, clamped, and seeked
to `_viewLeftMs` instead of the tapped position.

Deleted `_pxToMs` entirely. The body's `LayoutBuilder` already owns
`pxPerMs`; now it's threaded directly into the tap handler:

```dart
onTapDown: (d) => _handleBodyTap(d.localPosition, pxPerMs),
```

**Files**: `lib/presentation/widgets/video_dub_timeline.dart`.

## Intent comment on `onDeleteClip`

The video-dub `deleteTimelineClip(id)` call deliberately does *not*
`File.delete(...)` — the startup scan flags orphans, and the same media
may be referenced by other rows (re-import, cue-generated). A future
reader would "helpfully" add a file delete without realising the reuse
case, so the intent is now anchored in a comment next to the call site.

**Files**: `lib/presentation/screens/video_dub_screen.dart`.

# Part 2 — Premiere-style track restructure

## Why

The old layout had TTS cues pinned to "A1" by convention, two toggleable
lanes (V2, A3) that defaulted to hidden, and every import eagerly
copied into `voice_asset/video_dub/{slug}/assets/`. The result looked
less like Premiere and more like a single-purpose dubbing strip. The
user's ask was: default 5 tracks, Premiere semantics, external-path
imports (don't force files into the app's folder), A1 linked to V1's
audio with the ability to split/delete.

## Schema v14 — `linkGroupId`

`TimelineClips` gained one nullable column:

```dart
// Premiere-style link group: V1 video + A1 video-audio clips share the
// same id so a drag/trim on V1 can move its A1 sibling in lock-step.
// Null = not linked.
TextColumn get linkGroupId => text().nullable()();
```

Migration uses the repo's existing drop-and-recreate pattern (dev-mode;
video-dub data loss on upgrade is acceptable at this stage). Drift
regenerated `app_database.g.dart` via
`flutter pub run build_runner build --delete-conflicting-outputs`.

**Files**: `lib/data/database/tables.dart`,
`lib/data/database/app_database.dart`,
`lib/data/database/app_database.g.dart` (generated).

## New lane constants

```dart
class DubLanes {
  static const int v2 = -2;  // images — render above V1
  static const int v1 = -1;  // video clips
  static const int a1 =  1;  // V1's audio (linked)
  static const int a3 =  3;  // imported audio (SFX, music)
}
```

Lane index `2` is reserved for the day A2 (TTS cues) ever migrates into
`TimelineClips`. Until then, A2 is rendered directly from
`SubtitleCues` and has no `TimelineClip` rows.

## `_TrackKind` enum rename

Old: `{ v2, v1, a1, a2, a3 }` where `a1` meant TTS and `a2` meant
imported audio.

New: `{ v1, v2, a1, tts, a3 }`. `a1` now means "V1's audio" (lane
+1, sourceType `video-audio`); `tts` is the SubtitleCues-backed row at
"A2 · TTS"; `a3` is imported audio at lane +3.

`_TrackRow` factories rearranged — `tts(...)` stays, new
`videoAudio(...)` added, and the `removable:` parameter is gone
everywhere. All five rows are `const`, built once per frame via
`_buildTrackList()`.

The `_showV2` / `_showA3` bools, the `Tracks ▾` popup, the
`_onTrackMenuSelected` handler, and the per-header hide icon are all
deleted.

**Files**: `lib/presentation/widgets/video_dub_timeline.dart`.

## Import flow — external paths, no copy

`_importMedia` used to:

```dart
final destPath = PathService.dedupeFilename(assetsDir, ...);
await File(src).copy(destPath);
// …then insert one row pointing at destPath
```

Now it:

- Stores the absolute source path directly (`audioPath: src` — column
  name still says "audio" pending the rename that's piggybacked on
  the drag/trim PR).
- Probes duration via `FFmpegService.probeDurationSeconds` for video
  and audio imports. Image imports still default to 3 s.
- For a **video** import, inserts **two** rows in the same transaction:
  a V1 row (lane -1, `sourceType: 'video'`) and an A1 row (lane +1,
  `sourceType: 'video-audio'`), both carrying the same `mediaPath`,
  start, duration, and a freshly-generated `linkGroupId`.
- For image and audio imports, inserts a single row on V2 or A3
  respectively.

Result: projects become portable project files pointing at external
media, Premiere-style, at the cost of the usual "missing media"
failure mode if the user moves the source file later (rendered red via
`clip.missing`, though see the rough-edges note about flipping that
flag).

The helper `_ensureAssetsDir` is gone. The TTS-cue audio generated by
the software still lives in `voice_asset/video_dub/{slug}/…` — that's
managed content, not imported, and stays unchanged.

**Files**: `lib/presentation/screens/video_dub_screen.dart`,
`lib/presentation/widgets/video_dub_timeline.dart`
(`laneAndSourceForImport` remapped, `makeDubClipCompanion` gained
`linkGroupId`).

## A1-gated V1 audio playback

V1 video plays through media_kit; its audio is inside the mp4
container. To express "deleting an A1 clip silences that part of V1",
the tick handler checks A1 coverage each frame and mutes `_player`
when no A1 clip covers the playhead:

```dart
void _applyA1Gating(int ms) {
  // …walk TimelineClips, find any with laneIndex == DubLanes.a1
  //    whose [startTimeMs, startTimeMs + durationSec*1000) contains ms
  final covered = …;
  if (covered == _a1Covers) return;        // latched — no-op
  _a1Covers = covered;
  final target = (_muteVideoAudio || !covered) ? 0.0 : 100.0;
  _player.setVolume(target);
}
```

- The `_muteVideoAudio` toggle is an **override on top** — if the user
  muted V1 explicitly, it stays muted regardless of A1.
- `_a1Covers` is a latch so `setVolume` only fires on
  coverage-transitions (not every ~33 ms).
- Toggling `_muteVideoAudio` manually invalidates the latch
  (`_a1Covers = null`) and re-applies gating immediately, so the
  volume flip takes effect even while the video is paused.

Called from `_onVideoTick(pos)` at the top of the tick body — always,
regardless of cue state.

**Files**: `lib/presentation/screens/video_dub_screen.dart`.

## Waveform moved to the A1 lane

The waveform used to render as a background behind the TTS cues (old
A1 semantics). Under the new model that's visually dishonest — the
waveform is V1's audio envelope, not the TTS track. Moved it into
`_buildClipLane` under `_TrackKind.a1` so it paints behind the
video-audio clips it actually represents.

The old overlay in `_buildCueLane` is gone.

**Files**: `lib/presentation/widgets/video_dub_timeline.dart`.

## Import menu labels

The single `Import ▾` popup (already unified pre-session) now reads:

- Video → V1 (+ A1 audio)
- Image → V2
- Audio → A3

Reflects where each import actually lands under the new model.
