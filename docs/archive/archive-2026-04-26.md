# Part 1 — Initial four follow-ups

## Replace "Change Video" → "Export Video"

The top-bar button switched from "Open / Change Video" (a file picker
that wrote `project.videoPath`) to **Export Video**. Clicking runs
FFmpeg to mux V1 video + every generated TTS cue + every A3 imported
audio clip into one MP4 at a user-picked path. Original V1 audio is
included unless `_muteVideoAudio` is on.

The picker side is gone because importing a video via the timeline
now sets `project.videoPath` (see next section), so the dedicated
button became redundant. The empty-state video surface lost its
"Open Video" button too — it now just tells the user to import via
the timeline.

The export builds an `adelay` per overlay then `amix=…:duration=first
:dropout_transition=0[aout]`, mapped onto `0:v` with `-c:v copy`.
That `duration=first` choice broke things — fixed in Part 2.

**Files**: `lib/presentation/screens/video_dub_screen.dart`.

## Timeline V1 import → main video surface

The previous restructure decoupled `TimelineClips` rows from the
media_kit player — V1 imports landed on the lane but the central
surface stayed empty (`project.videoPath` was only set by the now-gone
"Open Video" picker). `_importMedia` now writes the imported source
back to `project.videoPath` in the same transaction that inserts the
V1 + A1 rows, so the surface picks it up via the existing
`_syncVideo(project)` post-frame callback.

**Files**: `lib/presentation/screens/video_dub_screen.dart`.

## Batch subtitle + TTS export

New folder-zip icon in the subtitle panel header. Picks a destination
folder, then writes:

- `<project>.srt` — all cues with HH:MM:SS,mmm timing.
- `tts/cue_001_mm-ss-mmm.<ext>`, `cue_002_…`, … — one file per
  cue with audio, copied from the cue's `audioPath`.
- `manifest.tsv` — `start_ms<TAB>end_ms<TAB>file<TAB>text` so a
  consumer can re-time without re-parsing the SRT.

Lets the user feed Premiere/DaVinci/Audacity directly. Skipped cues
without audio are counted in the snackbar.

**Files**: `lib/presentation/screens/video_dub_screen.dart`.

## A2 cue blocks become draggable

Each cue block on the A2 lane now accepts horizontal drag. While the
drag is in flight, `_dragCueId` + `_dragCueDxPx` shift the block's
`Positioned.left`; on drag end the screen receives
`onMoveCue(cue, newStartMs)` and persists via `updateSubtitleCue`,
preserving duration (only `startMs`/`endMs` shift, both by the same
delta). Drag is disabled when `onMoveCue` is null so the widget stays
optional-callback safe. Live `pxPerMs` is threaded in, so the
behaviour scales correctly when the range scrubber is zoomed.

No snap, no overlap-prevention.

**Files**: `lib/presentation/widgets/video_dub_timeline.dart`,
`lib/presentation/screens/video_dub_screen.dart`.

## Export-Video truncation bug

User reported a 10-minute video exporting as 2 seconds. Root cause:

```text
amix=…:duration=first    +  -shortest
```

`duration=first` makes amix stop when the **first input in the mix
list** ends — and the first input was the first TTS cue (e.g. 2 s
long). `-shortest` then truncated the muxed video to the audio rail.

Fix: `duration=longest` and drop `-shortest` entirely. The video copy
(`-c:v copy` on `0:v`) drives length; audio just plays as long as it
has content. `adelay` now uses three values (`d|d|d`) to cover the
rare 3-channel case ffmpeg can produce.

**Files**: `lib/presentation/screens/video_dub_screen.dart`.

## Export Audio (audio-only WAV)

+ every generated TTS cue + every A3 import onto a single 16-bit PCM

the user mux the dubbed audio over the original video in another
tool — the user's stated workaround for not having full Premiere-class
muxing in-app.

**Files**: `lib/presentation/screens/video_dub_screen.dart`.

# Part 3 — sidecar SRT + subtitle pane reshuffle

## Export Video writes a sidecar `.srt` by default

After the FFmpeg mux succeeds, the same project's cues get serialised
to SRT and written next to the MP4. Path derivation is
`_replaceExtension(outPath, '.srt')` so `myproject_dubbed.mp4` gets
`myproject_dubbed.srt`. Sidecar over `-c:s mov_text` because the
sidecar is universally readable + doesn't lock exports to MP4 if the
user types another extension into the save dialog.

The SRT writer (`_cuesToSrt`) is reused by `_exportSubtitlesAndTts`
so both code paths emit identical timestamps.

**Files**: `lib/presentation/screens/video_dub_screen.dart`.

## Subtitle pane restructure

- Top of the pane now hosts a full-width **Export TTS Audio + SRT**
  button — the headline action, distinct from the (then-)top-bar
  Export Audio (mixed WAV).
- Header row trimmed to `Subtitles · Add cue · Clear`. The folder-zip
  icon and the standalone Sync icon are gone.
- Footer stacks **Sync cue lengths to TTS** above **Generate All**,
  both full-width.

**Files**: `lib/presentation/screens/video_dub_screen.dart`.

## Auto-TTS + auto-sync switches in the SRT-import dialog

The dialog now returns `({bool replace, bool autoTts, bool autoSync})`
instead of a bare bool. Two `SwitchListTile`s control:

- **Auto-generate TTS after import** — runs Generate All immediately.
- **Auto-sync cue lengths to audio** — snaps `endMs` to
  `startMs + audioDuration*1000` for every cue with audio.

State is sticky for the editor session (`_autoTtsAfterImport`,
`_autoSyncAfterImport`) so toggling once persists across both bulk
import and the later Add-cue path (Part 5).

`_generateAll` was split into two methods:
- `_generateAll` — the user-facing entry point with the
  Skip/Regenerate/Cancel confirm dialog.
- `_runGenerateAll` — the loop only, called by both `_generateAll`
  and the auto-flow (which doesn't show a second dialog after the
  user already opted in via the switch).

**Files**: `lib/presentation/screens/video_dub_screen.dart`.

# Part 4 — Save UX + progress + export prefs

## Save no longer auto-exits

The old `_close` method bumped `updatedAt` and immediately called
`widget.onClose()` — clicking Save kicked you back to the project
list. Split into:

- **`_save`** — bumps `updatedAt`, clears `_dirty`, snackbar
  "Saved", **stays in the editor**.
- **`_back`** — if `_dirty`, prompts with **Cancel / Don't save /
  Save & Exit**; if clean, just closes.

Dirty state is tracked via a `_markDirty()` helper sprinkled into
every mutation path (imports, cue add/edit/delete/move/voice-change,
generate, sync, clip delete, etc.). Title bar shows a `• ` prefix
when dirty.

Edits already persist to the DB on the spot, so "Don't save"
really means "don't bump updatedAt" — wording was kept simple
because that nuance doesn't matter to users.

**Files**: `lib/presentation/screens/video_dub_screen.dart`.

## ffmpeg progress + success + open-folder

New shared widget `lib/presentation/widgets/export_progress.dart`
with three exports:

- **`runFfmpegWithProgress`** — runs ffmpeg with `-progress pipe:1
  -nostats`, parses `out_time_us=` lines into a `ValueNotifier<double>`,
  renders a modal `_ProgressDialog` (linear bar + % + Cancel button).
  Cancel kills the child process; on failure, the last 4 stderr
  lines are bubbled back to the caller for a snackbar.
- **`showExportSuccessDialog`** — "Export successful" with the file
  path (selectable, monospace) + an optional note. Two actions:
  **Open folder** (reveals in OS file manager) and **Done**.
- **`revealInFileManager`** — Windows `explorer /select,`, macOS
  `open -R`, Linux `xdg-open` on the parent dir. **Known bug**:
  passes `'/select,'` and the path as separate argv items, which
  Process.run on Windows joins with a space — the space breaks the
  flag and explorer opens at default location. Fix is `'/select,$path'`
  as one arg. Slated for follow-up.

Wired into both `_exportVideo` and `_exportAudio`. Total duration
is taken from the live media_kit `_duration` for video; for audio
it's the longest overlay end. Falls back to indeterminate when
unknown.

**Files**: `lib/presentation/widgets/export_progress.dart` (new),
`lib/presentation/screens/video_dub_screen.dart`.

## Truncation bug squashed (already in Part 2, mentioning here for completeness)

`amix duration=first` + `-shortest` truncated multi-minute videos
to the first cue's length. Now `duration=longest`, no `-shortest`,
video copy drives length.

## Export prefs (audio format + video/audio codec)

New `lib/data/storage/export_prefs.dart` wrapping `AppSettings`
key/value rows:

- **Audio format**: mp3 (default) / wav / flac — drives both file
  extension and `-c:a` for the audio export.
- **Video codec**: copy (default) / h264 / h265 / av1 — drives `-c:v`
  for the muxed export. `copy` keeps the existing fast/lossless
  behavior.
- **Video audio codec**: aac (default) / mp3 / opus — drives `-c:a`
  for the muxed export.

New Settings card (`_ExportPrefsCard`) in `settings_screen.dart`
renders a card with three Dropdowns. Auto-saves on each change.
Codec detection (`ffmpeg -codecs` parsing) was deliberately skipped
— heavy to parse and the user can spot a missing encoder from the
export error tail.

`_buildExportArgs` and `_buildAudioExportArgs` gained `videoCodec` /
`audioCodec` parameters; the export functions load the prefs via
`ref.read(exportPrefsServiceProvider).load()` on each click (one
DB round-trip, immeasurable).

**Files**: `lib/data/storage/export_prefs.dart` (new),
`lib/providers/app_providers.dart` (provider),
`lib/presentation/screens/settings_screen.dart`,
`lib/presentation/screens/video_dub_screen.dart`.

## "Manual generate first" bug — provider stream race

User reported: with auto-TTS toggled on in the Add-cue dialog, the
**first** cue added in a session never auto-generates; you have to
manually click the wand on any cue once, then auto-TTS works for
subsequent cues.

Root cause: `_generateOne` did

```dart
final providers = ref.read(ttsProvidersStreamProvider).valueOrNull ?? [];
```

`ttsProvidersStreamProvider` is a `StreamProvider` that's **not
watched** anywhere on this screen. `ref.read` lazily subscribes but
the first call returns `null` before the stream emits — `providers`
becomes `[]`, the `providerMap` lookup misses, and `_generateOne`
silently bails. Manual generation kicks the stream warm; subsequent
auto-TTS finds providers fine.

Fix: read providers straight from the DB.

```dart
final providers = await database.getAllProviders();
```

Same call the API server already uses. No more warm-up requirement.

**Files**: `lib/presentation/screens/video_dub_screen.dart`.

## Add-cue auto-flow (extending Part 3's import auto-flow to single cues)

`_addCueDialog` now passes `showAutoSwitches: true` to the cue
dialog (defaulting from the sticky session prefs), then chains:

1. Insert the cue with the chosen voice.
2. Re-read the row from the DB so `_generateOne` has a real
   `SubtitleCue` (not just a companion).
3. If auto-TTS opted in: call `_generateOne` for the new cue.
4. If auto-sync opted in: re-read post-generation and snap `endMs`
   to `startMs + audioDuration*1000`.

Same chain shape as the SRT-import auto-flow, just for one cue.

**Files**: `lib/presentation/screens/video_dub_screen.dart`.
